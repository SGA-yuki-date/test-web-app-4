# ユーザー基本情報登録 要件

## 概要

ログイン済みのユーザーが、健康記録・健康状態分析に必要な基本情報（生年月日・性別）を登録できる機能を提供する。基本情報が未登録の場合は登録画面へ誘導する。

## 受け入れ条件

- AC-001: ログイン後、ユーザー基本情報が未登録の場合にユーザー基本情報登録画面へ自動遷移または誘導されること
- AC-002: ユーザー基本情報登録画面に「生年月日 (yyyy/mm/dd)」入力フィールド、「性別」入力フィールド（または選択UI）、および「登録」ボタンが表示されること
- AC-003: ユーザーが生年月日と性別を入力して登録ボタンを押下すると、ユーザーの基本情報がデータベースに保存されること

## 機能要件

### 登録画面の表示制御

- ログイン済みセッションにおいてユーザー基本情報の登録状態を確認すること
- 未登録の場合、ユーザー基本情報登録画面を表示すること
- 登録済みの場合は登録画面を再表示しないこと

### 入力フィールド

- 生年月日は `yyyy/mm/dd` 形式で入力または選択できること
- 生年月日の形式・範囲バリデーションを行うこと
- 性別は選択式（例: 男性 / 女性 / その他）で入力できること

### データ保存

- 登録された生年月日・性別をログイン中のユーザーに紐付けて保存すること
- 登録完了後は次の画面（ホーム画面等）へ遷移すること
