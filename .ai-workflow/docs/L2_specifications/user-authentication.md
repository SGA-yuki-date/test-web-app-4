# ユーザー認証 要件

## 概要

メールアドレスとパスワードを用いたユーザー登録およびログイン機能をWebアプリとして提供する。

## 受け入れ条件

- AC-001: `/register` 等のURLでユーザー登録画面がWebページとして表示されること
- AC-002: ユーザー登録画面に「メールアドレス」入力フィールド、「パスワード」入力フィールド、および「登録」ボタンが表示されること
- AC-003: ユーザーがメールアドレスとパスワードを入力して登録ボタンを押下すると、ユーザーアカウントが新規作成されること
- AC-004: `/login` 等のURLでユーザーログイン画面がWebページとして表示されること
- AC-005: ユーザーログイン画面に「メールアドレス」入力フィールド、「パスワード」入力フィールド、および「ログイン」ボタンが表示されること
- AC-006: ユーザーが登録済みのメールアドレスとパスワードを入力してログインボタンを押下すると、認証が成功しログイン状態となること

## 機能要件

### ユーザー登録

- メールアドレスの形式バリデーションを行うこと
- パスワードは安全なハッシュ形式で保存すること
- 既に登録済みのメールアドレスで登録しようとした場合、エラーメッセージを表示すること

### ユーザーログイン

- 入力されたメールアドレスとパスワードをデータベースの登録情報と照合すること
- 認証成功時はセッションまたはトークンを発行しログイン状態を維持すること
- 認証失敗時はエラーメッセージを表示すること
